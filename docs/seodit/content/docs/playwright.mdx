## Matchers

Extend local `expect` once per spec file or shared fixture:

```typescript
import { test, expect as baseExpect } from "@playwright/test";
import { extendSeoditExpect } from "@sargonpiraev/seodit/playwright";

const expect = extendSeoditExpect(baseExpect);
```

Common matchers:

```typescript
await expect(page).toHaveMetaTitle("Anidex");
await expect(page).toHaveCanonical("https://anidex.tv/en");
await expect(page).toHaveSelfAlternate("en", "https://anidex.tv/en");
await expect(page).toHaveAlternate("ru", "https://anidex.tv/ru");
await expect(page).toHaveJsonLdType("WebSite");
```

## Spec naming

Use URL-based test titles and keep URL helpers local to the spec — avoid importing app navigation internals that pull env validation.

```typescript
test("/en", async ({ page }) => {
  await page.goto("/en");
  await expect(page).toHaveCanonical(`${process.env.NEXT_PUBLIC_SITE_URL}/en`);
});
```
